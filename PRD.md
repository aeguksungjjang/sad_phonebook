# PRD: 웹 기반 전화번호 관리 시스템 (Sad Phonebook)

## 1. 개요

간단한 전화번호 관리를 위한 웹 애플리케이션. 별도 로그인 없이 누구나 전화번호를 등록·수정·삭제할 수 있다.

---

## 2. 목표

- 브라우저에서 전화번호부를 간편하게 관리한다.
- 서버리스 인프라(Supabase + Vercel)로 운영 비용을 최소화한다.

---

## 3. 범위 (Scope)

### 포함
- 연락처 목록 조회
- 연락처 추가 (이름, 전화번호)
- 연락처 수정
- 연락처 삭제

### 제외
- 회원가입 / 로그인 / 인증
- 연락처 그룹·태그 분류
- 검색·필터 기능
- 연락처 가져오기/내보내기

---

## 4. 사용자 스토리

| ID | 스토리 |
|----|--------|
| US-01 | 사용자는 전체 연락처 목록을 볼 수 있다. |
| US-02 | 사용자는 이름과 전화번호를 입력하여 새 연락처를 추가할 수 있다. |
| US-03 | 사용자는 기존 연락처의 이름 또는 전화번호를 수정할 수 있다. |
| US-04 | 사용자는 연락처를 삭제할 수 있다. |

---

## 5. 기능 요구사항

### 5.1 연락처 목록 (Read)
- 페이지 로드 시 Supabase에서 전체 연락처를 조회하여 표시한다.
- 표시 항목: 이름, 전화번호, 수정 버튼, 삭제 버튼

### 5.2 연락처 추가 (Create)
- 이름(필수), 전화번호(필수) 입력 폼을 제공한다.
- 폼 제출 시 Supabase `contacts` 테이블에 행을 삽입한다.
- 추가 성공 후 목록을 즉시 갱신한다.

### 5.3 연락처 수정 (Update)
- 목록에서 수정 버튼 클릭 시 해당 항목을 편집 가능 상태로 전환한다.
- 변경 사항 저장 시 Supabase에서 해당 행을 UPDATE한다.

### 5.4 연락처 삭제 (Delete)
- 삭제 버튼 클릭 시 확인 없이 즉시 삭제한다. (기본 구현 우선)
- Supabase에서 해당 행을 DELETE한다.

## 8. 데이터 모델

### `contacts` 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid (PK) | 자동 생성 |
| `name` | text | 연락처 이름 |
| `phone` | text | 전화번호 |
| `created_at` | timestamptz | 생성 시각 (자동) |
